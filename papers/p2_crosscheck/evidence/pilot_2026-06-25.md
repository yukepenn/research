# P2 CrossCheck — pilot evidence (2026-06-25)

Real models via subscription CLIs (no API cost): author/reviewer ∈ {`claude`, `codex` (gpt-5.5)},
bidirectional. 10 controlled defect cases × 2 author→reviewer pairs × 3 workflows. 60 episodes in
the immutable ledger (paper_id=p2_crosscheck). Final correctness = deterministic hidden test.

## Result — final-patch correctness by workflow (matched budget = 2 model calls)
| workflow | final correct | n |
|---|---|---|
| NO_REVIEW (author once) | 0.80 | 20 |
| AUTHOR_MORE_COMPUTE (author self-improves) | 0.80 | 20 |
| CROSS_FAMILY_REVIEW (other model repairs) | 0.80 | 20 |

Identical for both authors (Claude 0.80 / Codex 0.80 in every workflow).

## Per-defect breakdown — the result is a CEILING + two benchmark bugs
8 of 10 defect types are solved one-shot (1.00) by both models under every workflow. The only two
non-1.00 cells — `type_serialization` and `collateral_regression` — failed 0.00 under ALL workflows,
and on inspection both were **underspecified benchmark bugs, not model failures**:
- `type_serialization`: the spec said "a list" but the hidden test required a *sorted* list — the
  model cannot know to sort. **Fixed** (spec now states "sorted ascending"); Claude then solves it.
- `collateral_regression`: the spec ("normalize() silently changes dedup semantics") did not state
  that `normalize` must strip-only / not lowercase. **Fixed**; Claude then solves it.
Both fixes verified: with fair specs, both cases are now solved one-shot (manual 5.7: fix the
benchmark, never charge the model). The two cases are excluded from the pilot's workflow analysis
(`exclusions.csv`).

## Reading (honest — this is the weakest of the three pilots)
- **H2 holds only trivially and is underpowered.** Cross-family review gives NO net benefit over
  giving the same author equal extra compute — but both equal *no review* too (all 0.80), so there is
  no regime in this corpus where workflows differ. After fixing the two benchmark bugs the corpus is
  effectively all-ceiling: strong models one-shot these small, well-specified defects, leaving no
  headroom to measure review value.
- **No complementarity is observable** at this difficulty (the same hard cases — when they existed —
  failed under every workflow; once fixed, everything passes).
- **Diagnosis / next step (the real pilot value):** measuring when a second model is worth it
  REQUIRES a genuinely mid-difficulty, fairly-specified defect corpus where NO_REVIEW lands around
  0.4–0.6 (e.g., the natural agent near-miss corpus on real repositories, harder multi-file defects).
  The budget-matched protocol, the author-extra-compute arm, the complementarity model, and the
  ReviewRoute router (validated on simulated data) are ready; the differentiating empirical study is
  gated on that harder corpus.

## Status implication
P2-H2 → PARTIAL (no differentiation observed; underpowered by an easy corpus). P2's current
contributions are the methodology (budget-matched protocol + complementarity + router) plus the
honest finding that strong models one-shot easy controlled defects, which sets the difficulty bar
for the full study.
