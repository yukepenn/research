# ToolMorph (p1_toolmorph) — Nearest-Neighbor Novelty Matrix

Generated 2026-06-25 by the Literature Scout. Every paper below was fetched and verified on its real venue/arXiv page (`verified=true` in `related_work.csv`).

**Our central claim (restated for comparison).** Holding the underlying tool STATE TRANSITION identical and changing only the model-visible interface representation significantly changes agent success, error types, cost, and even model RANKING; a semantic normalization layer that (a) does NOT depend on transform labels and (b) works on held-out tools/transforms reduces this sensitivity.

**Our four protected novel cores (the only things we claim):**
1. **EQ** — interface equivalence DEFINED by environment state-transition equality.
2. **META** — metamorphic PAIRED evaluation over behavior-preserving interface transforms.
3. **RANK** — model ranking stability / reversal caused by representation alone.
4. **NORM** — automatic semantic normal form, label-free, transferring to UNSEEN transforms/tools.

In the matrix, each cell marks whether the neighbor addresses that dimension: `=` matches/overlaps ours, `~` partial/adjacent, `x` absent. "Overlap-with-ours" gives the single largest collision.

## Where our distinct contribution survives (prose)

The closest prior art clusters into three groups, and crucially **no single paper combines our four cores**. (1) Schema/representation transformers — **TSCG** and **Learning to Rewrite Tool Descriptions** — already do automatic, generalizing interface transforms, but they optimize for token cost / performance via fixed operators or learned rewriting, both of which sit inside our declared novelty boundary; neither defines interface equivalence by state-transition equality, neither runs a metamorphic paired contrast, and neither studies ranking. (2) Robustness/reliability evaluators — **ReliabilityBench**, **Schema First**, **On the Robustness of Agentic Function Calling**, **ToolSandbox**, **tau-bench** — supply the pieces we will reuse (end/final-state oracles, pass^k, stateful sandboxes, controlled interface variation) but each perturbs the wrong variable for our claim: they change the task, inject faults, expand toolkits, or vary spec STYLE, rather than applying behavior-preserving transforms to a tool whose state transition is held byte-identical. (3) Ranking-fragility papers — **When Benchmarks are Targets**, **AgentAtlas**, and the side-finding in **Natural Language Tools** — establish that measurement/format choices reorder models, but for QA prompts, evaluation protocols, or a one-shot JSON→NL swap, not for a metamorphic family of state-equivalent tool interfaces.

The single highest-risk collision is **ReliabilityBench**: it literally uses the words "metamorphic relations" and scores by "end-state equivalence," overlapping META and EQ at the vocabulary level. Our defense must be sharp and is real: ReliabilityBench perturbs tasks and injects faults (including schema *drift* as a *failure*), which CHANGE the underlying state transition; ToolMorph instead defines equivalence classes of interfaces that induce the SAME state transition and varies only the model-visible representation. The second-highest risk is the **NORM** cluster (TSCG + Rewrite-Tool-Desc), where "automatic interface transform that generalizes" is already demonstrated; our surviving delta is a *canonical, transform-label-independent normal form that transfers to UNSEEN transform families* (not merely unseen tools, and not a fixed operator set or a performance-tuned rewriter).

**Recommended framing to protect novelty:** lead with EQ+META as an inseparable unit ("equivalence defined by state-transition equality, evaluated by paired metamorphic contrast over representation-only transforms") because that combination is unoccupied; treat RANK as the headline empirical finding while explicitly citing n05/n11/n12 so reviewers see we extend, not rediscover, ranking fragility; and scope NORM as *label-free transfer to held-out transforms*, contrasting directly against TSCG (fixed operators) and Rewrite-Tool-Desc (learned, tool-level, performance-objective) to avoid a "deterministic schema compilation already does this" rejection.
